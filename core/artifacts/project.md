## ✅ 문서 완성도 체크리스트

> ⚠️ 모든 필수 항목이 체크되어야 Frozen 상태로 전환 가능

### 필수 섹션 (Required)
- [ ] 프로젝트 규모 정의 완료 (S/M/L)
- [ ] 기술 스택 전체 항목 결정됨
- [ ] 프로젝트 구조 정의됨
- [ ] 명명 규칙 정의됨
- [ ] 품질 기준 정의됨

### 품질 체크 (Quality)
- [ ] 모든 기술 선택에 근거 명시됨
- [ ] 버전 번호가 구체적으로 명시됨 (예: "1.x" 금지, "1.21.0" 필요)
- [ ] 금지 사항이 명확히 정의됨

### 연결성 체크 (Linkage)
- [ ] architecture-options.md의 선택이 반영됨
- [ ] plan.md의 비기능 요구사항과 일관성 확인됨

## ⚠️ Frozen 문서 변경 절차 (RFC)

> 이 문서가 Frozen 상태가 된 후에는 아래 절차로만 변경 가능

### RFC 절차
1. `rfc/` 디렉토리에 변경 제안서 작성
   - 파일명: `RFC-NNNN-<title>.md`
2. RFC 문서에 포함할 내용:
   - 변경 사유
   - 영향 범위 분석
   - 대안 검토
   - 롤백 계획
3. Manager 승인
4. decision.md에 기록
5. project.md 업데이트

### RFC 없이 변경 가능한 항목
- 오타 수정
- 예시 추가 (규칙 변경 없이)
- 설명 보완 (의미 변경 없이)
