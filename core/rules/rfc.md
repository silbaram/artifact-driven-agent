# RFC (Request for Change) 규칙

> Frozen 문서 변경을 위한 공식 절차

## RFC가 필요한 경우

- project.md 변경
- plan.md의 확정된 기능 범위 변경
- 승인된 decision.md 항목 번복

## RFC 절차

1. RFC 문서 작성 (`rfc/RFC-NNNN-title.md`)
2. Manager 리뷰
3. 승인/거부 결정
4. 승인 시: 문서 업데이트 + decision.md 기록

## RFC 상태

| 상태 | 설명 |
|------|------|
| Draft | 작성 중 |
| Under Review | 리뷰 중 |
| Approved | 승인됨 |
| Rejected | 거부됨 |

## 긴급 변경

- 보안 취약점, 심각한 버그 등
- 구두 승인 후 적용
- 24시간 내 RFC 사후 작성
